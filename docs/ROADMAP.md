# ZiPPaY Roadmap

Planned enhancements and future milestones for the ZiPPaY ecosystem.

## ✅ Phase 1: Core UX (Completed)
- [x] **Smartwatch Micro-payments**: Basic circular UI and payment flow.
- [x] **Offline Mode**: Local storage based pending sync system.
- [x] **Transaction History**: List view of all recent payments.
- [x] **Sorting Controls**: Date and Amount based sorting for history.
- [x] **Limit Feedback**: Visual warnings for loading > ₹500.
- [x] **Spending Analytics**: 6-type interactive chart engine.
- [x] **AI Assistant**: Gemini-powered finance coach integration.

## 🚀 Phase 2: Security & Control (Upcoming)
- [ ] **Daily Spending Limits**: User-defined cap on total watch transactions per 24 hours.
- [ ] **Biometric Lock**: Require phone fingerprint/face ID to enable "Active Mode" on the watch.
- [ ] **Merchant Verification**: Trusted merchant list for repetitive low-value payments.

## 📊 Phase 3: Analytics & Engagement
- [ ] **Spending Categories**: Automatic categorization of merchant payments (Food, Transport, Shopping).
- [ ] **Budget Notifications**: Phone alerts when 80% of the daily limit is reached.
- [ ] **Rewards Program**: "ZiP Points" for consistent offline syncing.

## 🔗 Phase 4: Advanced Connectivity
- [ ] **NFC Proximity Simulation**: Require the "Merchant" and "Watch" tabs to be in interaction state.
- [ ] **Bluetooth API Integration**: Use the Web Bluetooth API for actual device-to-device communication testing.
